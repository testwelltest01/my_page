# 03. Node.js란 무엇인가?

앞서 배운 `npm`, `npx`, 그리고 여러 개발 도구(Vite, NestJS 등)를 이해하기 위한 **가장 근본적인 뿌리**인 Node.js에 대해 알아봅니다.

---

## 1. Node.js의 정의
**"자바스크립트(JavaScript)를 브라우저 밖에서도 실행할 수 있게 해주는 환경(Runtime)"**입니다.

*   **과거**: 자바스크립트는 크롬이나 사파리 같은 **웹 브라우저 내부**에서만 살 수 있었습니다. (웹사이트 버튼 클릭, 애니메이션 등 담당)
*   **현재 (Node.js의 탄생)**: 자바스크립트를 브라우저에서 꺼내서 **내 컴퓨터(OS)**에서 직접 실행할 수 있게 되었습니다. 
*   **결과**: 자바스크립트 하나만 공부해도 웹사이트뿐만 아니라 **서버(Backend)**, **스마트폰 앱(App)**, **PC 프로그램**까지 모두 만들 수 있게 된 것입니다.

---

## 2. 우리가 배운 내용과의 관계 (중요!)

Node.js는 모든 개발 도구들이 돌아가는 **'기초 토양'**과 같습니다.

### ① Node.js와 npm/npx
*   **Node.js**를 설치하면 **npm**과 **npx**가 함께 보너스로 설치됩니다.
*   Node.js가 **'운영체제'** 같은 느낌이라면, npm은 그 안에서 물건을 사고 관리하는 **'앱스토어'**입니다.

### ② 02번 파일(분야별 도구) 및 뼈대/재료와의 관계
우리가 정리했던 `create`(뼈대)와 `install`(재료) 도구들이 왜 모두 Node.js와 연결될까요?

*   **웹 (Vite, Next.js)**: 웹사이트 결과물은 브라우저에서 돌지만, 그 결과물을 **만들고 빌드하는 과정**은 내 컴퓨터의 Node.js 환경에서 일어납니다.
*   **앱 (Expo, React Native)**: 자바스크립트 코드를 실제 앱 파일로 변환해주는 도구가 Node.js 위에서 돌아갑니다.
*   **백엔드 (NestJS, Express)**: 서버 자체가 Node.js 환경에서 직접 실행됩니다. (브라우저 없이 내 컴퓨터/서버에서 24시간 돌아감)

---

## 3. 한눈에 보는 비유

| 항목 | 비유 | 설명 |
| :--- | :--- | :--- |
| **JavaScript** | **자동차** | 우리가 실제로 운전하는(코딩하는) 언어 |
| **Node.js** | **엔진 & 도로** | 자동차가 달릴 수 있게 해주는 핵심 장치와 환경 |
| **npm** | **부품 센터 / 튜닝샵** | 자동차에 필요한 타이어, 부스터(라이브러리)를 사는 곳 |
| **Vite / Next.js** | **전문 제조 공장** | 특정 용도(웹용)에 맞는 자동차 뼈대를 찍어내는 곳 |

---

## 💡 요약
사용자님이 지금 터미널에 입력하신 `npm create vite`라는 명령어가 작동하는 이유도, 사용자님의 컴퓨터에 **Node.js라는 엔진**이 이미 깔려있기 때문입니다. 

Node.js는 단순히 "서버 만드는 도구"를 넘어, **"자바스크립트로 세상을 만드는 모든 과정의 기본 환경"**이라고 이해하시면 완벽합니다!
