# 06. 브라우저의 작동 원리와 DOM (어떻게 화면이 그려질까?)

우리가 주소창에 URL을 입력하고 엔터를 치는 순간부터 화면에 글자가 보이고 버튼이 작동하기까지, 브라우저 내부에서는 아주 복잡한 요리가 시작됩니다. 그 과정을 자세히 알아봅니다.

---

## 1. 브라우저의 핵심 3단계: 파싱(Parsing) -> 렌더링(Rendering) -> 실행(Execution)

웹사이트는 결국 **텍스트(HTML, CSS, JS)**로 되어 있습니다. 브라우저는 이 텍스트를 읽어서 우리가 보는 화면으로 바꿉니다.

### ① HTML을 DOM 트리로 만들기 (Parsing)
브라우저는 HTML 파일의 텍스트를 한 줄 한 줄 읽으면서 **DOM(Document Object Model)**이라는 나무 모양의 구조를 만듭니다.
- **HTML**: "안녕"이라는 글자가 써진 종이
- **DOM**: 그 종이를 브라우저가 이해하기 쉽게 **객체(Object)**라는 부품으로 조립한 상태

### ② CSS를 CSSOM 트리로 만들기
HTML을 읽는 도중 스타일 시트(CSS)를 만나면, 각 요소에 어떤 디자인이 입혀질지 계산하여 **CSSOM**이라는 또 다른 지도를 만듭니다.

### ③ 렌더 트리(Render Tree)와 배치(Layout)
DOM(내용)과 CSSOM(디자인)을 합쳐서 실제로 화면에 보일 것들만 추려낸 **렌더 트리**를 만듭니다. 그 후, 각 요소가 화면의 **어느 위치(x, y)**에, **어떤 크기**로 보일지 계산합니다. (이 과정을 **Layout** 혹은 **Reflow**라고 합니다.)

### ④ 그리기 (Paint)
드디어 계산된 수치대로 화면에 픽셀을 채워 넣습니다. (이 과정을 **Paint**라고 합니다.)

---

## 2. DOM(Document Object Model)이란 무엇인가?

가장 중요한 개념입니다! **DOM은 자바스크립트가 HTML을 조작할 수 있게 해주는 연결 다리**입니다.

*   **왜 필요한가?**: 자바스크립트는 '언어'일 뿐, HTML을 직접 만질 줄 모릅니다. 브라우저가 HTML을 **자바스크립트가 알아들을 수 있는 객체 형태(DOM)**로 바꿔줘야 비로소 자바스크립트가 "저 버튼 색깔 바꿔줘!"라고 명령할 수 있습니다.
*   **구조**: 부모-자식 관계를 가진 트리(Tree) 구조입니다. 
    - `<body>`가 아빠라면, 그 안의 `<div>`는 자식인 셈이죠.

---

## 3. 자바스크립트의 등판 (JS Execution)

브라우저가 HTML을 읽다가 `<script>` 태그를 만나면 어떻게 될까요?

1.  **멈춤 (Blocking)**: 브라우저는 HTML 읽기를 잠시 멈추고 자바스크립트 파일을 먼저 가져와서 실행합니다. (그래서 JS 파일이 너무 크면 화면이 늦게 뜹니다!)
2.  **조작**: 실행된 자바스크립트는 브라우저가 만들어 놓은 **DOM을 직접 건드립니다.** 
    - `document.getElementById('btn')` 같은 명령어가 바로 "DOM에서 ID가 btn인 부품을 찾아와!"라는 뜻입니다.
3.  **다시 그리기**: 자바스크립트가 DOM을 수정하면(예: 글자 변경), 브라우저는 다시 **Layout -> Paint** 과정을 거쳐 화면을 업데이트합니다.

---

## 4. 리액트(React)가 여기서 중요한 이유 (Virtual DOM)

사용자님이 방금 만든 프로젝트는 **React**입니다. 리액트는 왜 유명해졌을까요?

*   **문제**: 위의 3번 과정(다시 그리기)은 컴퓨터에게 아주 힘든 일입니다. 자바스크립트가 수시로 화면을 바꾸면 브라우저가 계속 Layout, Paint를 반복하느라 느려집니다.
*   **해결 (Virtual DOM)**: 리액트는 가상의 DOM을 메모리에 하나 더 만들어둡니다.
    1.  자바스크립트로 화면을 바꾸면 진짜 DOM이 아니라 **가상 DOM**에서 먼저 바꾼 뒤,
    2.  진짜 바뀐 부분만 계산해서 **진짜 DOM에 딱 한 번** 반영합니다.
*   **효과**: 훨씬 빠르고 효율적으로 화면을 그릴 수 있게 됩니다.

---

## 5. 한눈에 보는 요약도

| 단계 | 명칭 | 비유 |
| :--- | :--- | :--- |
| **1단계** | **DOM 생성** | 설계도를 바탕으로 부품 조립하기 |
| **2단계** | **CSSOM 생성** | 부품별 색칠 가이드 만들기 |
| **3단계** | **Layout** | 부품을 어디에 놓을지 배치하기 |
| **4단계** | **Paint** | 실제로 화면에 예쁘게 그리기 |
| **5단계** | **JS 실행** | 사용자의 클릭에 따라 부품을 교체하거나 숨기기 |
