# npm vs npx 핵심 요약

`npm`과 `npx`는 모두 Node.js 생태계에서 중요한 도구이지만, 그 용도와 작동 방식에는 확실한 차이가 있습니다.

## 🚀 한 줄 요약

> **"npm은 패키지를 '설치(Manage)'하고, npx는 패키지를 '실행(Execute)'합니다."**

---

## 1. 주요 차이점 비교

| 구분            | npm (Node Package Manager)                  | npx (Node Package Execute)                    |
| :-------------- | :------------------------------------------ | :-------------------------------------------- |
| **주요 역할**   | 패키지 설치, 버전 관리, 의존성 관리         | 패키지 실행 (설치 없이 바로 실행 가능)        |
| **설치 여부**   | 로컬(`node_modules`) 또는 전역(`-g`)에 설치 | 설치하지 않고 임시로 내려받아 실행 후 삭제    |
| **사용 사례**   | 라이브러리 추가, `package.json` 관리        | `create-react-app` 같은 일회성 실행 도구 사용 |
| **명령어 예시** | `npm install lodash`                        | `npx create-react-app my-app`                 |

---

## 2. npm (Node Package Manager)

**npm**은 이름 그대로 패키지 매니저입니다. 프로젝트에 필요한 라이브러리를 다운로드하고, `package.json` 파일을 통해 이들의 버전을 관리하는 데 사용됩니다.

- **저장**: 패키지를 컴퓨터나 프로젝트 폴더에 실제로 저장합니다.
- **관리**: 프로젝트를 다시 빌드하거나 다른 개발자와 공유할 때 필요한 모든 도구를 준비해 줍니다.
- **비유**: 가전제품을 사서 집에 들여놓는 것과 같습니다. 내 공간을 차지하지만 언제든 쓸 수 있습니다.

## 3. npx (Node Package Runner)

**npx**는 패키지를 실행하는 데 특화되어 있습니다.

- **일회성**: 도구를 전역(`-g`)으로 설치하고 싶지 않을 때 사용합니다. (예: 최신 버전의 도구를 딱 한 번만 쓰고 싶을 때)
- **편의성**: 프로젝트 내 로컬에 설치된 패키지를 실행할 때 경로(`node_modules/.bin`)를 직접 찾을 필요 없이 `npx [명령어]`로 바로 실행할 수 있습니다.
- **비유**: 넷플릭스 스트리밍과 같습니다. 굳이 소장(설치)하지 않아도 필요할 때 바로 꺼내 볼 수 있습니다.

---

## 4. 언제 무엇을 쓸까요?

- 프로젝트에서 계속 사용할 라이브러리(React, axios 등)를 추가할 때는 **`npm install`**을 사용하세요.
- 컴퓨터에 흔적을 남기고 싶지 않거나, 특정 도구의 최신 버전을 한 번만 실행하고 싶을 때는 **`npx`**를 사용하세요.

### 꿀팁

- **`npm install --`**을 너무 많이 해서 `node_modules` 폴더가 너무 커졌다면, 그 폴더만 지우고 나중에 다시 쓸 때 용량이 적은 `package.json`만 있는 상태에서 **`npm install`**을 다시 치면 모든 재료가 부활합니다!
